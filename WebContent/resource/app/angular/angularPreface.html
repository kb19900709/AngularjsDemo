傳統網站


如果一個 page (html、JSP ... etc)，涵蓋了大量的 javaScript & jQuery 控制

以及該頁面必須依賴資料狀態被重複使用多次

會發生什麼事情呢 ?

  • 維護成本提高
  • 擴充成本提高
  • 可讀性降低
  • Ajax & CallBack Hell

傳統網站網頁控制大部分針對 DOM (Document Object Model) 的操作

使用 javaScript & jQuery 的解決方案可以解決大部分的問題

但極有可能在專案日趨龐大時遇到上述4點需要克服




SPA

 

Single Page Application 以下簡稱SPA

  • 傳統WEB : Multi - Page
  • SPA : Single - Page 

 

優點

  • 類 Desktop Application ，使用者體驗較佳
  • 使用過的資源無須重新載入，資源快取由瀏覽器處理
  • 前後端關注點分離。前端即是應用程式，後端負責計算資料，移植性強
  • Server 無須再負責渲染HTML。僅需提供資料輸出入，減輕負載效能獲得提升

缺點

  • 不利於 SEO  
  • 瀏覽器歷史紀錄需實作
  • 第一次載入應用程式耗時較久
