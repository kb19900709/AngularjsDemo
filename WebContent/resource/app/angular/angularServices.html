注入


一個使用陣列語法，一個沒有

兩種語法效果一樣

第一種寫法

javaScript在經過壓縮處理之後

註解會被丟棄、空格會被刪除、變數會被重新命名

angularjs 會失智找不到對應的service


第二種寫法

安全式注入 angularjs 會去解析字串名稱

並找到對應的service(依陣列裡的順序)

因此推薦使用第二種寫法


通訊

  • $http : angularjs 負責產生請求的 service
  • ngResource : angularjs 負責產生RESRful url pattern 請求的選擇性模組
  • Promise物件 : javaScript 處理非同步請求的標準，依賴連結與成功錯誤處理器
